# Procedimientos y Duración Variable de Turnos

## Resumen de Cambios

Este documento describe las nuevas funcionalidades agregadas al sistema de turnos médicos.

## 1. Entidad Appointment - Campo `duration_minutes`

### Cambios en la Base de Datos
Se agregó el campo `duration_minutes` a la tabla `appointments` con valor por defecto de 30 minutos.

```sql
ALTER TABLE appointments ADD COLUMN duration_minutes INT DEFAULT 30;
```

### Uso
Ahora al crear un turno se puede especificar la duración en minutos:
- Si no se especifica, usa el valor del tenant (`appointmentDurationMinutes`)
- Si se especifica `procedureId`, usa la duración del procedimiento
- Si se especifica `durationMinutes` explícitamente, tiene prioridad sobre todo lo demás

## 2. Nueva Entidad Procedure

### Estructura de la Tabla
```sql
CREATE TABLE procedures (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    tenant_id UUID NOT NULL,
    specialty_id BIGINT,
    name VARCHAR(200) NOT NULL,
    duration_minutes INT NOT NULL DEFAULT 30,
    active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### Endpoints CRUD

| Método | Endpoint | Descripción | Roles |
|--------|----------|-------------|-------|
| GET | `/procedures` | Listar todos los procedimientos | OWNER, STAFF, PROFESSIONAL |
| GET | `/procedures/{id}` | Obtener procedimiento por ID | OWNER, STAFF, PROFESSIONAL |
| GET | `/procedures/by-specialty/{specialtyId}` | Listar por especialidad | OWNER, STAFF, PROFESSIONAL |
| POST | `/procedures` | Crear procedimiento | OWNER, STAFF, PROFESSIONAL |
| PUT | `/procedures/{id}` | Actualizar procedimiento | OWNER, STAFF, PROFESSIONAL |
| DELETE | `/procedures/{id}` | Eliminar procedimiento (soft delete) | OWNER, STAFF, PROFESSIONAL |

## 5. Validación de Solapamiento de Turnos

La lógica de validación ahora considera el rango completo del turno `[start, start + duration]`:

- Un nuevo turno **NO** puede crearse si existe solapamiento con otro turno existente
- El sistema detecta cuando:
  - El inicio del nuevo turno es antes del fin del existente
  - Y el fin del nuevo turno es después del inicio del existente

### Ejemplo
Si existe un turno de 10:00 a 10:45:
- ❌ No se puede crear un turno a las 10:30 (solapamiento)
- ❌ No se puede crear un turno a las 9:30 con duración de 60 min (solapamiento)
- ✅ Se puede crear un turno a las 10:45 (sin solapamiento)
